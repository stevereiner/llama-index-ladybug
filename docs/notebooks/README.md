# Notebooks

Two example notebooks are provided for using Ladybug with LlamaIndex.

---

## `property_graph_ladybug.ipynb` — Property Graph (recommended)

Uses the modern `PropertyGraphIndex` API with `LadybugPropertyGraphStore`.

**Covers:**
- Creating a structured property graph with a custom relationship schema
- Extracting triples from documents using `SchemaLLMPathExtractor`
- Enabling and querying a vector index on chunk nodes
- Building a combined graph + vector retriever
- Running a `RetrieverQueryEngine` over the combined results

Install: `uv pip install llama-index-ladybug llama-index-llms-openai llama-index-embeddings-openai`

---

## `LadybugGraphDemo.ipynb` — Knowledge Graph (legacy)

Uses the older `KnowledgeGraphIndex` API with `LadybugGraphStore`.

**Covers:**
- Building a knowledge graph from unstructured text
- Querying with and without source text context
- Hybrid query mode with embeddings
- Visualizing the graph with [pyvis](https://pyvis.readthedocs.io/) → `ladybuggraph_draw.html`
- Manually adding triplets

This API is not deprecated by llama-index and remains fully functional. Use it if you prefer the simpler triplet-based graph model.

Install: `uv pip install llama-index-ladybug llama-index-llms-openai pyvis`

---

## `ladybuggraph_draw.html`

A pre-rendered interactive graph visualization generated by the `LadybugGraphDemo` notebook using pyvis. Open in any browser to explore the graph.
